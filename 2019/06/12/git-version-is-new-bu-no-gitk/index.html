git版本不老，但是没有gitk，kali系统

六月 12, 2019
  • git
  • kali

学习git时，遇到没有gitk，kali系统。以下是解决这个问题的一些记录。

提示信息

bash: gitk: command not found
 

我的解决方法是2,直接安装gitk。

 

解决方法1：更新git。

这个方法的第一步就遇到了问题
提示信息

aptsources.distro.NoDistroTemplateException: Error: could not find a distribution template for Kali/n/a

找不到kali的分发模板

百度到的解决方案，在sources.list里添加更新源 （但是我还是没解决）

deb http://ppa.launchpad.net/yubico/stable/ubuntu cosmic main
deb-src http://ppa.launchpad.net/yubico/stable/ubuntu cosmic main

而找不到分发模板的意义我不确定是git的开发者未提供问题还是ppa。找到一篇14年的英文文章，似乎是kali Linux 需要添加PPA repository

Kali Linux add PPA repository add-apt-repository

按着步骤走下来（第一步没完成，提示没有这个包的安装，而我感觉自己的电脑可能已经装了所以没有在意了。），ppa确实可以添加和更新了（这里遇到的一个细节是在原文图里也有注明，./add-apt-repository ppa:git-core/ppa ，因为我即使已经在目录下，用add-apt-repository ppa:git-core/ppa ，得到的提示信息还是原来的，没有改变。）。但还是没法更新到官网上的最新版。
而且我发现提示已经是最新的版本2.20.1了，可是我看官网上最新的是2.22.0。


我觉得已经够了，可能是对于kali来说2.20.1就已经是最新的了。就没再深入。（而且问题已经由方法2解决）

 

解决方法2：直接安装gitk
